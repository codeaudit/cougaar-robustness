Configuring the Defense Coordinator under ACME and Cougaar 11.2
 
Overlays
The Coordinator is provided in two overlays:

Robustness Telcordia believability/coordinator HEAD B11_2
Robustness Objs Coordinator HEAD B11_2

Because the Coordinator depends on the CommunityStatusModel,
which is populated by MIC's ARUC1 defense, the following
overlays are also required:

Robustness Objs sensors HEAD B11_2
Robustness Objs Common HEAD B11_2
Robustness Boeing ExNihilo HEAD B11_2
Robustness MIC Mgmt Agent HEAD B11_2
Robustness Robustness common HEAD B11_2
Rules

Assuming all the overlays above have been loaded, the Coordinator requires that the following  rules be applied:

  - $CIP/csmart/config/rules/robustness
  - $CIP/csmart/config/rules/robustness/common
  - $CIP/csmart/config/rules/coordinator
  - $CIP/csmart/config/rules/coordinator/test
  - $CIP/csmart/config/rules/robustness/uc1
  - $CIP/csmart/config/rules/metrics/basic
  - $CIP/csmart/config/rules/metrics/sensors
  - $CIP/csmart/config/rules/metrics/serialization/metrics-only-serialization.rule
  - $CIP/csmart/config/rules/metrics/rss/tic
- community_rules:
  - $CIP/csmart/config/rules/robustness/communities

Note: Only the two below are specific to the Coordinator.  The others are for ARUC1.


$CIP/csmart/config/rules/coordinator - This rule loads the Coordinator's plugins into
                                       each ARManager and a subset into each Node.  

$CIP/csmart/config/rules/coordinator/test - This optional rule loads several monitor plugins
                                            and servlets useful for testing and debugging defenses.

Include Scripts

Include the following include scripts are provided:

$CIP/csmart/lib/unleash_defenses.rb - This script Unleashes Defenses at :during_Stage_1.
                                      It should be loaded in all Saves with the
                                      Coordinator loaded.

$CIP/csmart/lib/leash_on_restart.rb - This script Leashes Defenses on a Restore
                                      at :snapshot_restored, and then Unleashes
                                      them at :starting_stage.  It should be loaded
                                      in all Restores with the Coordinator loaded.


Actions and States
The Coodinator defines the following ACME Actions and States for Leashing and Unleashing
Defenses in the following file:

$CIP/csmart/config/lib/coordinator/leashing.rb
# This action is used to confirm that Defenses have been Unleashed by all Robustness Managers.
# The argument specifies the level of verbosity: 0 is silent(the default), 1 prints the message.
do_action "MonitorUnleash", 1
Starting: MonitorUnleash
Finished: MonitorUnleash in 0 seconds
...
 INFO: All Robustness Managers have Unleashed Defenses.

# This action Unleashes all Defenses so that the Coordinator can enable them and they can
# operate normally.  Its intended for execution after startup.  If its not done, the
# Defenses will never be enabled by the Coordinator.
# The argument specifies the level of verbosity: 0=silent(the default), 1=info, 2=debug.
do_action "Unleash", 1
Starting: Unleash
 INFO: Requested that CONUS-ARManager Unleash Defenses.
 INFO: Requested that REAR-ARManager Unleash Defenses.
 INFO: Requested that 1-AD-DIV-ARManager Unleash Defenses.
 INFO: Requested that UA-ARManager Unleash Defenses.
Finished: Unleash in 0 seconds

# This action Leashes Defenses when restarting.  Its intended to be used when restoring a full
# society from persistence.
do_action "LeashOnRestart"

# This state is reached when All the Robustness Managers have Unleashed their Defenses.
# Defense Actions will not be enabled until after this point, so no use starting them.
# The first argument specifies the level of verbosity: 0=silent(the default), 1=info.
# The second argument is a timeout.
wait_for "Unleashed", 1, 3.minutes
Waiting for: Unleashed
 INFO: CONUS-ROB-MGMT-NODE: Defenses Unleashed.
 INFO: REAR-ROB-MGMT-NODE: Defenses Unleashed.
 INFO: 1-AD-DIV-ROB-MGMT-NODE: Defenses Unleashed.
 INFO: UA-ROB-MGMT-NODE: Defenses Unleashed.
Done: Unleashed in 3 seconds
